## “Problem” section (satirical)

**Every other debugger waits for the crime. We’ve moved on.**

Current tools (including Replay’s own, in the joke) **require the bug to have already happened**: write code → ship → break → record → replay → root cause. The page calls building only a backward time machine “embarrassing” and says **Replay Precog goes the other direction** (fictionally).

---

## “How Precog works” — three fictional capability cards

Headline: **How Precog works.**  
Subhead: **One new MCP tool. Three capabilities that shouldn’t be possible.**

1. **Speculative execution analysis** — Claims a speculative React reconciler across a “probabilistic AST shadow DOM” and sample quotes about future race conditions / naming mistakes.  
2. **Your personal bug fingerprint** — Claims inferring mistakes from typing velocity, naming habits, “Redux anti-pattern fingerprint,” etc.  
3. **Pre-generated recordings** — Claims generating a full Replay recording **before** the bug occurs.

Treat these as **in-universe marketing**, not engineering specs.

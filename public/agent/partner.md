## Your agent is only as good as what it can see

When something breaks at runtime — flaky test, race condition, production-only bug — agents are **flying blind** without DOM, network timing, or component tree: **codebase + guess.**

**Replay MCP** gives browser runtime context so agents can **diagnose precisely and fix the first time.** The program looks for teams who want to make that experience as useful as possible.

## What design partners get

Intro: **This isn’t a beta waitlist. It’s a working relationship.**

1. **Free for the full engagement** — Full Replay MCP access at **no cost** for the partnership: **minimum 30 days, up to three months**. No credit card. Use on real work.

2. **Direct line to the engineering team** — Access to Replay engineering in **Discord** (questions, feedback, when something doesn’t match expectations).

3. **Your workflow shapes the product** — Occasional calls on how you use Replay MCP (what works, what’s painful); feedback influences what gets built next.

4. **Preferred pricing when you’re ready** — At the end of the engagement, friendly pricing reflecting the relationship; design partners “won’t be treated like strangers at renewal” (site copy).

---

## Who this is for

**Replay MCP is purpose-built for debugging React-based web apps.** If the team has meaningful **React** surface area (product, platform, internal tool) and uses AI coding agents (**Cursor, Claude Code, Codex, Copilot, Windsurf**), the program is designed for you.

If the stack is **primarily mobile, backend, or outside React**, the page says **Replay MCP probably isn’t the right fit yet** — they’d rather say that upfront than waste your time.

Beyond stack: you care about **shortening the debugging loop** and will **say honestly** when something doesn’t work as expected.
